* **Ren’Pyプロジェクト内に `doc` / `docs` / `manual` フォルダを置くことは問題ありません**
* **エンジン動作・ビルド・配布にも影響しません**
* **目的に応じて置き場所を分けるのが理想的**です

---

## Ren’Pyのフォルダ構造の前提（大事なポイント）

Ren’Pyは基本的に：

* **実行時に参照するのは `game/` フォルダの中だけ**
* `game/` の外にあるファイルは、
  👉 **ゲームには読み込まれないが、プロジェクトとしては自由に置ける**

という構造になっています。

---

## おすすめの置き場所（目的別）

### ① 制作メモ・設計書・仕様書（開発者向け）

**👉 プロジェクト直下が最適**

```text
my_project/
├─ game/
├─ doc/
│  ├─ design.md
│  ├─ scenario_notes.txt
│  ├─ ui_memo.md
│  └─ license_notes.md
├─ README.md
└─ renpy.exe
```

**理由**

* Ren’Pyに一切干渉しない
* GitHubにもそのまま載せやすい
* 「これは開発用資料だ」と一目でわかる

📌 フォルダ名の定番

* `doc` / `docs`（一番よく使われる）
* `manual`（操作説明書的なもの向け）
* `notes`（思考メモ寄り）

---

### ② プレイヤー向けマニュアル（同梱したい場合）

#### A. ゲーム内で読ませない場合（PDF・txtとして同梱）

```text
my_project/
├─ game/
├─ manual/
│  └─ how_to_play.txt
```

✔ 配布ZIPにそのまま入る
✔ ゲーム起動とは完全に独立

---

#### B. ゲーム内で表示したい場合（Ren’Pyで読む）

この場合だけ注意が必要です。

```text
game/
├─ manual/
│  └─ how_to_play.txt
```

* `game/` 内に置く必要あり
* `renpy.file()` や `renpy.open_file()` で読む

ただし、

> **Ren’Pyは txt / md を直接「画面表示用」として読むのが少し面倒**

なので、
**日本のノベルゲームでは「マニュアル専用ラベルを書く」方が一般的**です。

---

## `.md`（Markdown）についての注意

* Ren’Pyは **Markdownを解釈しません**
* `.md` はあくまで
  👉 **人間（開発者・GitHub）向け**

なので、

| 用途     | OK？    |
| ------ | ------ |
| 開発メモ   | ◎      |
| 設計書    | ◎      |
| README | ◎      |
| ゲーム内表示 | ❌（非推奨） |

## 個人的におすすめの構成（かなり安定します）

```text
my_project/
├─ game/
│  ├─ script.rpy
│  ├─ animations.rpy
│  └─ ui.rpy
├─ docs/
│  ├─ 00_overview.md
│  ├─ 10_scenario.md
│  ├─ 20_ui_design.md
│  ├─ 30_state_notes.md
│  └─ 99_memo.txt
├─ README.md
└─ LICENSE.txt
```

👉 **あなたの制作スタイル（整理・後から見返す・公開も意識）にとても合います**
