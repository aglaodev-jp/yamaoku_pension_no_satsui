# サモエドルートの大浴場の選択肢

① **唯が見た人影のことを考える**（客室の選択肢で”２．「ベッドどっちにする？僕、窓際がいい！」僕は一緒になってはしゃいだ。”を選んだ場合に増える）
② **今回の映画の撮影について考える**（必ず出る選択肢）
③ **唯のことを考える**（必ず出る選択肢）

はい、その整理はかなり良くなっています。
前よりずっと **「木下ルートの成立条件」と「サモエドルート内の差分条件」** がはっきりしました。

特に良いのは、

* **木下ルート確定条件** を2つの通過に絞ったこと
* **サモエドルート側の選択肢フラグを減らしたこと**

です。
この2つで、実装も確認もだいぶ楽になります。

---

# 今回の整理を言い換えると

## 木下ルート確定条件

以下の2つを両方通過したら木下ルート：

* 客室で
  **「ベッドどっちにする？僕、窓際がいい！」**
* ロビーで
  **「ここは木下さんに相談だ」**

つまりこれは **AND条件** です。

---

## サモエドルート側

大浴場の選択肢は3つのうち、

* ①は条件付き
* ②③は常時表示

この形ですね。

これはとても自然です。
プレイヤーにとっても「選択肢が少なすぎる」感じが出ませんし、
実装側でも分岐の事故が減ります。

---

# おすすめのフラグ構成

フラグ名は、意味が見えるものにすると後で助かります。

```renpy
# 客室
default flag_room_fun = False

# ロビー
default flag_lobby_kino = False

# ルート確定
default route_kino = False
default route_samoyed = False
```

---

# 実装イメージ

## 1. 客室の選択肢

```renpy
label room_choice:

    menu:
        "……ここで一緒に一夜を過ごすのか。僕はドギマギした。":
            jump room_choice_end

        "「ベッドどっちにする？僕、窓際がいい！」僕は一緒になってはしゃいだ。":
            $ flag_room_fun = True
            jump room_choice_end

        "部屋をひと通り見渡して、映研の僕は現実に引き戻された。":
            jump room_choice_end

label room_choice_end:
    return
```

---

## 2. ロビーの選択肢

ここで木下相談フラグを立てます。

```renpy
label lobby_choice:

    menu:
        "ペンションを探検しながら撮影しよう":
            jump lobby_explore

        "ここは木下さんに相談だ":
            $ flag_lobby_kino = True
            jump lobby_consult

        "……今日はもう休まない？":
            jump lobby_rest
```

---

## 3. ルート確定

ロビーのシーンが終わったあと、まとめて判定します。

```renpy
label route_check:

    if flag_room_fun and flag_lobby_kino:
        $ route_kino = True
        jump kino_route_start
    else:
        $ route_samoyed = True
        jump samoyed_route_start
```

---

# ここで大事な点

あなたの今の案だと、

* ロビーで「ここは木下さんに相談だ」を選んでも
* 客室ではしゃいでいなければ

**木下ルートには入らない**

という設計です。

これはまったく問題ありません。
ただ、ここは自分で明確に決めておくと安心です。

つまり仕様としては：

> 木下さんに相談しただけでは木下ルートにならない。
> 旅行っぽく少し浮かれた空気を通過した上で、さらに木下さんに相談したときだけ木下ルートになる。

この思想なら、非常に筋が通っています。

# サモエドルートの大浴場選択肢

今回の簡略化、とても良いです。
これなら扱いやすいです。

```renpy
label samoyed_bath_choice:

    menu:
        "唯が見た人影のことを考える" if flag_room_fun:
            jump think_shadow

        "今回の映画の撮影について考える":
            jump think_movie

        "唯のことを考える":
            jump think_yui
```

---

# この形の良いところ

## 1. 条件付き選択肢が1つだけ

管理がとても楽です。
デバッグもしやすいです。

## 2. 必ず2択以上になる

`flag_room_fun` が立っていなくても、

* 映画のこと
* 唯のこと

は必ず出るので、画面が寂しくなりません。

## 3. 客室での選択が後半にちゃんと効いている

これが大事です。
プレイヤーに「前の選択が生きている」感じが出ます。

---

# さらにきれいにするなら

`route_kino` と `route_samoyed` は、実はなくても動きます。

たとえばこうです。

```renpy
if flag_room_fun and flag_lobby_kino:
    jump kino_route_start
else:
    jump samoyed_route_start
```

これでも十分です。

ただし、後で

* 回想モード
* CG達成管理
* エンディング判定
* ログ確認

などに使う可能性があるなら、
ルートフラグを残しておくのは有効です。

その場合はこうしておくときれいです。

```renpy
label route_check:

    $ route_kino = False
    $ route_samoyed = False

    if flag_room_fun and flag_lobby_kino:
        $ route_kino = True
        jump kino_route_start
    else:
        $ route_samoyed = True
        jump samoyed_route_start
```

# ひとつだけ確認しておくとよい点

今回の条件だと、
客室②を選んでいてもロビー②を選ばなければサモエドルートです。

その場合、大浴場で

* 「唯が見た人影のことを考える」
* 「今回の映画の撮影について考える」
* 「唯のことを考える」

の3つが出る可能性があります。

これは仕様として自然です。
むしろ良いです。
「木下ルートには行かなかったが、客室での軽いはしゃぎは後に影響している」と見せられるからです。
